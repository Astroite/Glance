# Glance 入门指南

## 快速开始

### 1. 安装 Glance

从 Releases 页面下载最新的 Windows 安装包：

- **MSI 格式**: `Glance_0.1.0_x64.msi`
- **NSIS 格式**: `Glance_0.1.0_x64-setup.exe`

运行安装程序，按提示完成安装。

### 2. 首次启动

启动 Glance 后，您将看到图库设置界面：

1. 点击 **"添加图库"** 按钮
2. 选择包含照片的文件夹（本地或 NAS 路径）
3. 点击 **"开始扫描"**

扫描过程将：
- 遍历文件夹中的所有图片文件
- 计算文件身份（xxh3 哈希）
- 提取 EXIF 元数据
- 配对 RAW + JPEG 文件
- 生成三档缩略图

### 3. 浏览时间线

扫描完成后，照片将按拍摄日期分组显示：

- **瀑布流视图**: 自适应列数，适合浏览大量照片
- **网格视图**: 固定列数，适合快速预览
- 使用 **年/月导航** 快速跳转到特定时间段

### 4. 查看照片详情

点击任意照片打开大图预览：

- 显示完整照片（1080px 缩略图）
- 查看 EXIF 信息（相机、镜头、参数）
- 查看 XMP 评分和标签
- 查看关联文件状态（RAW、JPEG、XMP）

## 功能详解

### 文件身份识别

Glance 使用 `xxh3(head 64KB + tail 64KB + file_size)` 作为文件身份标识：

- **不依赖文件路径**: 移动文件后仍可识别
- **不依赖修改时间**: mtime 仅用于变更检测
- **支持大文件**: 只读取首尾各 64KB，速度快

### RAW + JPEG 配对

同一目录下、相同文件名的 RAW 和 JPEG 文件自动配对：

```
photos/
├── IMG_001.CR2    ← RAW 原片
├── IMG_001.JPG    ← 展示文件（缩略图来源）
└── IMG_001.XMP    ← XMP sidecar（评分/标签）
```

配对规则：
- JPEG 作为展示文件和缩略图来源
- RAW 作为关联原片
- XMP 中的 Rating 和 Label 合并到照片记录

### 缩略图系统

三档缩略图，WebP 格式：

| 档位 | 短边尺寸 | 用途 |
|---|---|---|
| Small | 240px | 网格视图 |
| Medium | 480px | 瀑布流视图 |
| Large | 1080px | 大图预览 |

特性：
- EXIF Orientation 自动烘焙
- 非 sRGB 色彩空间自动转换
- 缺失时显示占位图

### 缺失文件处理

当文件不可达时（NAS 断开、文件移动/删除）：

1. 标记为 `missing` 状态
2. 显示缓存的缩略图（如果有）
3. 显示 "Missing" 徽章
4. 需要用户手动重定位

**不会自动操作**：
- 不自动删除缩略图缓存
- 不自动分类为移动/重复/删除/离线
- 不修改用户原始文件

## 快捷键

| 快捷键 | 功能 |
|---|---|
| `Esc` | 关闭大图预览 |
| `←` / `→` | 上一张/下一张照片 |
| `F` | 全屏预览 |
| `G` | 切换网格/瀑布流 |

## 数据存储

所有数据存储在 `%APPDATA%/Glance/`：

```
%APPDATA%/Glance/
├── index.sqlite          # 主数据库
├── index.sqlite-wal      # WAL 日志
├── index.sqlite-shm      # 共享内存
├── thumbs/               # 缩略图缓存
│   ├── 240/
│   ├── 480/
│   └── 1080/
├── config.json           # 应用配置
└── logs/                 # 日志文件
```

## 常见问题

### 扫描很慢？

首次扫描需要：
- 遍历所有文件
- 计算哈希值
- 提取 EXIF
- 生成缩略图

后续扫描使用增量检测，只处理变更文件。

### 照片显示 "Missing"？

可能原因：
- NAS 未连接
- 文件被移动或删除
- 路径权限不足

解决方法：
1. 检查文件是否可访问
2. 使用 "重定位" 功能更新路径

### 缩略图模糊？

缩略图是预生成的 WebP 文件。如果原始文件更新，需要重新扫描以生成新缩略图。

### 如何添加多个图库？

在图库设置界面可以添加多个文件夹，每个文件夹独立管理。
