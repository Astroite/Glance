# Glance 掠影设计文档

## 1. 产品定位

**Glance 掠影** 是一款面向摄影用户的本地优先照片回顾应用。

它不替代 Lightroom、DxO 等专业修图软件，也不接管用户的 RAW 工作流。它的目标是提供一个轻量、现代、快速的照片浏览和回顾体验，让用户可以随时按时间线、地点、相机、镜头等维度重新浏览自己的照片 archive。

一句话定位：

> 一个本地优先、摄影师友好的现代照片时间线浏览器。

---

## 2. 核心问题

现有工具存在明显断层：

- 专业修图软件适合修图和出片，但不适合随意回顾大量历史照片。
- Immich / PhotoPrism 等工具体验不错，但需要服务端部署，对普通 Windows 用户有门槛。
- NAS 原生相册工具通常界面普通，摄影师工作流支持有限。
- 直接浏览 RAW 体验慢，且通常无法还原专业修图软件调整后的最终效果。

Glance 要解决的是：

> 用户已经有专业修图工作流，但缺少一个舒服、轻量、现代的本地照片回顾层。

---

## 3. 目标用户

主要目标用户：

- 使用 Lightroom / DxO 等专业修图软件的摄影爱好者或摄影师。
- 照片原片存放在 NAS、本地硬盘或移动硬盘中。
- 希望偶尔快速回顾过去照片，而不是每次打开专业修图软件。
- 重视本地隐私，不希望必须上传云端。
- 希望软件安装简单，不想部署 Docker、Postgres 或后台服务。

---

## 4. 设计原则

### 4.1 不接管工作流

Glance 不修改原始照片目录，不移动 RAW，不替代修图软件。

默认以只读方式扫描图库目录，不修改用户数据。

### 4.2 本地优先

索引、缩略图、元数据、搜索结果都存储在本机。

默认不依赖云服务，也不要求外网访问。

### 4.3 轻量现代

打开即用，界面接近现代相册体验：时间线、快速滚动、月份分组、大图预览、收藏、筛选，地图查看。

### 4.4 摄影师友好

支持相机、镜头、焦段、光圈、快门、ISO、拍摄时间、GPS 等摄影元数据筛选。

### 4.5 RAW 只做预览，不做修图

RAW 支持的目标是“能看”和“能关联”，不是还原专业调色。

---

## 5. 核心功能

### 5.1 时间线浏览

- 按年份、月份、日期组织照片。
- 支持快速跳转年份和月份。
- 支持瀑布流或网格视图。
- 缩略图懒加载，保证大图库滚动流畅。
- 前端使用虚拟滚动（virtual scrolling），10 万级图库下保持稳定帧率。

### 5.2 本地索引

- 首次选择照片目录后建立索引。
- 后续支持增量扫描。
- 索引数据存储在本地 SQLite。
- 应用关闭后无需后台服务器运行。

**文件身份策略：**

- 不以绝对路径作为主键。
- 使用 `content hash + size` 作为物理文件身份，路径仅作为可变定位信息。
- `mtime` 不参与身份判断，仅作为变更检测字段：当同一路径的 `mtime` 或 size 变化时，Glance 重新计算 hash 并判断文件是否已变更。
- Glance 不自动判定文件是移动、重复、删除还是离线；路径不可达或扫描未见时统一视为"缺失"，等待用户手动重定位。

**多图库结构预留：**

- 数据库 schema 从第一版起即包含 `library_id` 字段。
- MVP 默认单图库，结构上为后续多图库管理做好准备，避免后期重构。

**NAS / 离线鲁棒性：**

- 图库不可达、路径变化、文件移动、文件删除、重复文件等情况，在 UI 中统一表现为"原文件缺失"。
- 原文件缺失时不清空索引，不删除缩略图缓存；Glance 使用已缓存的预览图继续展示，并明确提示用户原文件缺失，需要手动重定位。
- SMB 路径变化或断连不应导致缩略图缓存被误清。

### 5.3 缩略图缓存

- 本地生成缩略图缓存。
- 缓存存放在用户 AppData 目录。
- 支持清理缓存和重建缓存。

**多档尺寸：**

- 网格小图：240px 短边。
- 时间线 / 单列：480px 短边。
- 大图预览：1080px 短边。
- MVP 首次索引时为每张照片生成并缓存全部三档，避免缩放伪影或大图模糊。
- 1080px 短边作为 MVP 大图预览上限。
- 缩略图缓存缺失且原文件暂不可用时，前端显示占位图。

**EXIF Orientation：**

- 缩略图生成时即按 Orientation 旋转烘焙。
- 前端无需再处理旋转，缩略图与大图朝向保持一致。

**色彩管理（ICC）：**

- 原图带 Adobe RGB / Display P3 等非 sRGB profile 时，缩略图统一转换为 sRGB 并嵌入 profile。
- WebView 默认按 sRGB 渲染，转换缺失会导致明显色偏。

**HEIC 支持：**

- 使用 libheif 自行解码，不依赖 Windows 系统 HEVC 扩展。
- 避免用户因缺失系统 codec 而无法浏览 iPhone 等设备产出的 HEIC 文件。

### 5.4 元数据读取

支持读取：

- 拍摄时间
- 相机品牌 / 型号
- 镜头
- 焦距
- 光圈
- 快门
- ISO
- 分辨率
- GPS
- 文件格式

**XMP sidecar：**

MVP 阶段至少读取 XMP sidecar 中的：

- `Rating`（星级，0–5）
- `Label`（颜色标签）

许多专业修图软件会把评分和标签信息写入 sidecar 中，不读取等于忽略用户已有的整理成果。

关键词、修图调整等更复杂字段排到 0.3。

### 5.5 搜索与筛选

基础筛选：

- 日期范围
- 相机
- 镜头
- 文件类型
- 评分 / 收藏
- 有无 GPS

后续可支持全文搜索和智能搜索。

**中文分词：**

SQLite FTS5 默认不支持中文分词。若开启全文搜索，需使用 `jieba-rs` 或类似分词器自定义 tokenizer。

### 5.6 RAW 预览

MVP 阶段不做完整 RAW 渲染。

处理策略：

1. JPEG / PNG / HEIC 等可直接浏览图片作为优先显示文件。
2. 同目录、同文件名 stem 的 RAW + JPEG 组合在 MVP 中视为同一张照片。
3. RAW + JPEG 组合中，JPEG 负责缩略图和大图预览，RAW 作为关联原片保留。
4. RAW-only 文件优先提取内嵌 JPEG 作为预览；不做 demosaic，不承诺还原修图效果。

---

## 6. 非目标

Glance 初期不做以下事情：

- 不替代 Lightroom / DxO 等专业修图软件。
- 不做专业 RAW 调色。
- 不做云同步。
- 不做多用户权限系统。
- 不做公网访问。
- 不强制运行后台服务。
- 不直接管理或移动用户原片目录。

---

## 7. 技术架构

推荐技术路线：

```text
前端：React / Svelte + TypeScript（虚拟滚动必备）
桌面壳：Tauri
核心：Rust
数据库：SQLite
搜索：SQLite FTS5（中文分词需自带 tokenizer，如 jieba-rs）
缩略图：本地 JPEG / WebP cache（三档尺寸 + sRGB + Orientation 烘焙）
HEIC：libheif（不依赖系统 codec）
RAW 预览：rawler（纯 Rust，优先提取内嵌 JPEG）
视频预览：FFmpeg，可选；为控制安装包体积，默认不内置，按需下载或调用系统 ffmpeg
```

基础架构：

```text
Glance.exe
├─ WebView UI
│  └─ 现代照片时间线界面
│
├─ Rust Core
│  ├─ 文件扫描
│  ├─ EXIF 读取
│  ├─ 缩略图生成
│  ├─ SQLite 查询
│  ├─ RAW 预览提取
│  └─ 后台任务队列
│
└─ Local Data
   ├─ index.sqlite
   ├─ thumbnails/
   └─ config.json
```

MVP 阶段建议不启动 localhost HTTP server，而是使用 Tauri command 让前端直接调用本地 Rust 方法。

只有未来需要手机端、局域网访问或多设备协同时，再考虑把核心抽成 HTTP API。

## 8. MVP 范围

第一版目标是做出一个稳定可用的本地照片回顾器。

MVP 功能：

- Windows 安装包。
- 添加本地或 NAS 照片目录。
- 建立本地索引（`hash + size` 作为文件身份，`mtime` 仅用于变更检测）。
- 读取 EXIF 拍摄时间及基础参数。
- 读取 XMP sidecar 中的 Rating / Label。
- 识别同目录、同文件名 stem 的 RAW + JPEG 组合，并在时间线中作为一张照片展示。
- 为每张照片生成 240 / 480 / 1080 三档缩略图（含 EXIF Orientation 烘焙、sRGB 色彩转换）。
- 按时间线浏览（虚拟滚动，支持瀑布流和网格两种视图）。
- 年 / 月快速跳转。
- 大图预览。
- 只读图库模式。
- 缺失文件处理（NAS 不可达、文件移动、删除、重复等情况统一提示原文件缺失，使用缓存预览，等待用户手动重定位）。

暂不进入 MVP（移至 0.2）：

- 收藏。
- 基础筛选 UI。

暂不进入 MVP（更远）：

- 人脸识别。
- AI 搜索。
- 手机端同步。
- 多用户。
- 完整 RAW 渲染。
- 自动云备份。

- 基础筛选 UI（相机 / 镜头 / ISO / 焦段 / 评分 / 有无 GPS）。

## 10. 主要风险

### 10.1 大图库性能

大量照片、NAS 慢连接和缩略图生成可能造成卡顿。需要做好任务队列、懒加载和缓存策略。

### 10.2 NAS 稳定性

SMB 目录可能断连、变慢或路径变化。MVP 统一以"原文件缺失"处理，不删除索引和缓存，并要求用户手动确认重定位。

### 10.3 RAW 兼容性

不同相机 RAW 格式差异较大，尤其是各家的压缩 RAW 格式。初期应避免承诺完整 RAW 渲染。

### 10.4 产品边界膨胀

如果加入同步、AI、多用户和移动端，产品会迅速变成完整服务器型相册系统。需要保持“本地照片回顾器”的核心定位。

---

## 11. 一句话总结

**Glance 掠影不是修图软件，也不是云相册。它是一个安装在 Windows 上、本地优先、摄影师友好的照片时间线回顾工具。**

它最重要的价值是：

> 不打扰现有工作流，只让用户更舒服地重新看见自己的照片。
